The Pipeline

From Raw Data to Predictions.

A reproducible, end-to-end workflow covering every stage: data ingestion, cleaning, exploratory analysis, feature selection, and model training and comparison.

01

Data Loading & Cleaning

Ingesting the UCI heart disease dataset and preparing it for analysis.

  • Load 303 patients × 36 features from CSV
  • Rename raw column codes to descriptive names (e.g. cp → Chest_Pain_Type)
  • Impute missing values — median for numerics, mode for categoricals
  • Identify and remove duplicate records
  • Generate summary statistics with df.describe()
02

Exploratory Data Analysis

Visualising distributions, clinical patterns, and relationships across all features.

  • Target distribution — Heart Disease Severity (0–4 levels)
  • Univariate histograms: Age, Cholesterol (≥240 flag), Resting BP, Max HR
  • Categorical bar charts: Gender, Chest Pain Type, ECG Results
  • Bivariate: features vs disease level across age groups
  • Scatter plots, 3D visualisation, and Pair Grid
03

Feature Engineering & Selection

Creating a lean, clinically meaningful feature set including engineered risk proxies.

  • Binary target: Heart_Disease_Diagnosis > 0 → 1 (165 disease, 138 healthy)
  • 14 features selected — clinical + 1 engineered
  • Engineered: Risk_Score
  • Pearson correlation with target computed and ranked
  • 80 / 20 stratified train-test split, StandardScaler applied
04

Model Training & Comparison

Five algorithms trained on the same split and evaluated on the same held-out test set.

  • Logistic Regression — linear baseline with feature scaling
  • Random Forest — 100-tree bagging ensemble
  • SVM (RBF kernel) — maximum-margin classifier
  • XGBoost & LightGBM — sequential gradient boosting
  • All models saved as .joblib and .pkl

🔍 Key Findings from EDA

  • Blood Flow Test (Thal) is the single strongest predictor across all models
  • Chest Pain Type and Blocked Vessels (CA) are the next most important
  • Max Heart Rate is negatively correlated — higher exercise HR is protective
  • Logistic Regression achieves the best ROC-AUC (0.952) despite its simplicity
  • Engineered features (risk_score, age_thalach_ratio) provide meaningful uplift
  • Train/test gap is minimal across all models — no significant overfitting

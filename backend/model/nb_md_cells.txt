# Part 7 ΓÇö Gradient Boosting (XGBoost & LightGBM)

**Why Gradient Boosting?** Builds trees sequentially ΓÇö each tree corrects the errors of the previous one. State-of-the-art on structured/tabular data. Built-in regularization prevents overfitting.

**Does not require feature scaling** ΓÇö uses raw `X_train` / `X_test`.

| | XGBoost | LightGBM |
|---|---|---|
| Tree growth | Depth-wise | Leaf-wise (faster) |
| Speed on large data | Moderate | Very fast |
| Memory usage | Higher | Lower |
